What changed in code
Core and runtime:
- Agent settings include web observer toggles and runtime-facing web observer configuration fields
- runtime includes ObserverAccessRegistry for token issue/resolve/revoke with bounded tombstone retention semantics for expired/revoked states
- observer token formatting is opaque and generated via OS-backed randomness, with redacted Debug/Display behavior

Web transport:
- new crate oxide-agent-transport-web added as optional read-only monitoring transport
- web monitor exposes health endpoint, token-scoped snapshot endpoint and token-scoped SSE events endpoint
- watch page is token-scoped, read-only and consumes snapshot plus live SSE events
- auth paths map observer token resolution errors into explicit access outcomes and avoid cache persistence for protected responses

Telegram transport and app wiring:
- Telegram runtime/context receives shared observer access registry via DI from bootstrap
- task status updates can include watch URL button only when web observer is configured, URL is renderable by Telegram and web monitor is actually ready
- guardrail copy explicitly states watch view is read-only, controls remain in Telegram, and link is expiring/shareable
- terminal task transitions revoke active observer links for the task
- web monitor startup failure degrades gracefully without taking down Telegram runtime

Operational configuration
Web observer is disabled by default and must be explicitly enabled:

Environment variables (.env):
- WEB_OBSERVER_ENABLED=true|false (default: false)
- WEB_OBSERVER_BASE_URL=https://your-domain.com (required for Telegram links)
- WEB_OBSERVER_BIND_ADDR=0.0.0.0:8080 (internal container bind)
- WEB_OBSERVER_TOKEN_TTL_SECS=900 (token lifetime, default 15 min)

Docker Compose:
- Port 8080 exposed for web observer
- All WEB_OBSERVER_* variables passed through from .env

Endpoints (when enabled):
- GET /health - health check
- GET /api/observer/{token}/snapshot - task snapshot JSON
- GET /api/observer/{token}/events - SSE live events
- GET /watch/{token} - HTML watch page (main user-facing route)

Security notes:
- Tokens are short-lived (configurable TTL, default 15 minutes)
- Watch links are read-only; no mutation operations exposed via web
- Token format is opaque (oa_ prefix + 32-byte hex) generated via OS CSPRNG
- Invalid/expired/revoked tokens return distinct 401/403 responses
