Current repo status at handoff
Working tree was clean after the latest Stage 4 docs update.

What changed in code
Core:
- task lifecycle now includes Stopped as a terminal state distinct from Cancelled
- stop signal, safe-point and stop-report contracts are formalized in the core task model
- snapshots now validate stop-report persistence invariants for stopped tasks

Runtime:
- task registry and executor now support stop_and_report without reusing hard cancellation semantics
- graceful stop can be honored at waiting-input and loop-boundary safe points with persisted partial reports
- recovery helpers clear stop-report payloads when coercing tasks into non-stopped terminal states
- task event delivery is no longer single-consumer only; broadcaster subscriptions support replay and bounded live fan-out
- late subscribers can rebuild state from persisted snapshot and event history, while terminal tasks stay replay-only after in-memory cleanup

Transport status
- Telegram transport still has the pre-Stage-4 cancel path only
- owner-only stop controls, live task status updates and terminal notifications are the remaining transport work in Slice 4.4
