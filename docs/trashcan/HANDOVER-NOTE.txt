Topic
Agent Mode memory compaction refactor and rollout.

Goal
Replace the old local memory auto-compaction with an orchestration-layer staged compaction pipeline for Agent Mode only.
Chat mode is intentionally out of scope.

Status
Stages 1 through 12 are complete.
The agreed scope is implemented.

Latest commit
29b84aa - test(agent): harden compaction lifecycle regressions

What is implemented
- Compaction moved out of memory.rs into crates/oxide-agent-core/src/agent/compaction/
- Layered pipeline: budget estimation, classifier, externalization, pruning, summary compaction, rebuild, archive extension points, runner integration, transport UX, hardening/tests
- Separate compaction model with deterministic fallback
- Archive extension points with noop sink and archive refs
- Manual compact action for Agent Mode transport
- Progress lifecycle events and repeated compaction warning

Key guarantees now covered
- Pinned context is preserved
- topic AGENTS.md is preserved
- current task is preserved
- todos remain intact
- runtime injections and approval replay context are preserved
- recent raw user/assistant/tool window is preserved
- archive refs serialize even when archive sink is noop
- repeated compaction warning is emitted when multiple applied passes happen in one run
- invalid compaction-model JSON falls back safely
